# Brain — Второй мозг Егора

**Система агентов, состояние проекта, brain log и промпты команды.**

---

## Структура

```
data/brain/
├── agents.json          — метаданные 7 агентов (ключ, имя, роль, модель, промпт)
├── state.json           — текущее состояние проекта (цель, фокус, задачи, воронка)
├── log.jsonl            — brain log (решения, инсайты, что сработало / не сработало)
├── README.md            — этот файл
├── prompts/             — промпт-файлы агентов (источник истины для редактирования)
│   ├── chief.md         — Chief / Главный (OPUS)
│   ├── marketer.md      — Marketer / Маркетолог
│   ├── content_maker.md — Content Maker / Контент-мейкер
│   ├── analyst.md       — Analyst / Аналитик
│   └── copywriter.md    — Copywriter / Копирайтер
├── conversations/       — история диалогов с агентами
└── workflows/           — сохранённые workflow-сессии
```

## 5 агентов команды

| Агент | Ключ в системе | Модель | Что делает |
|-------|---------------|--------|------------|
| **Chief / Главный** | `ceo` | **Opus** | Единственная точка входа. Разбивает задачу, вызывает специалистов, логирует, собирает результат |
| **Marketer / Маркетолог** | `funnel` | Sonnet | Воронка, оффер, CTA, лид-магниты, стратегия контент-плана |
| **Content Maker / Контент-мейкер** | `content_strategy` | Sonnet | Готовые тексты каруселей, постов, Reels-скриптов, Stories в стиле Егора |
| **Analyst / Аналитик** | `analyst` | Sonnet | Разбор конкурентов, чужих воронок, трендов. Рентген механики |
| **Copywriter / Копирайтер** | `rewriter` | Sonnet | Переписывает чужой контент в стиль Егора. Анти-GPT и антиплагиат встроены |

Плюс 2 поддерживающих агента для workflow-цепочки:
- **Operations** (`operations`) — ежедневные задачи и порядок выполнения
- **Tech Architect** (`tech_architect`) — Cursor / root-fix задачи по Creovix

## Что нужно настроить

### Обязательно: переключить Chief на Opus

Chief (ключ `ceo`) использует переменную `OPENROUTER_AGENTS_MODEL_OPUS`.

Установить в секретах проекта:

```
OPENROUTER_AGENTS_MODEL_OPUS = anthropic/claude-opus-4
```

Если Opus 4 недоступен, альтернативы:
```
OPENROUTER_AGENTS_MODEL_OPUS = anthropic/claude-3-opus-20240229
OPENROUTER_AGENTS_MODEL_OPUS = anthropic/claude-opus-4-5
```

Остальные агенты используют:
```
OPENROUTER_AGENTS_MODEL_EXECUTOR = google/gemini-2.0-flash-001
```
(или любая быстрая Sonnet-уровня модель)

Также должен быть установлен:
```
OPENROUTER_API_KEY = <ваш ключ OpenRouter>
```

## Как применять промпты

Промпты хранятся в двух местах:
1. **`prompts/*.md`** — полные читаемые версии для редактирования человеком
2. **`agents.json`** — поле `systemPrompt` — рабочие версии, которые читает сервер

Если хочешь обновить промпт агента:
1. Отредактируй нужный `.md` файл в `prompts/`
2. Скопируй содержимое в поле `systemPrompt` в `agents.json` (одной строкой через `\n`)
3. Перезапусти workbench сервер

## Тестовое задание для проверки

После настройки `OPENROUTER_AGENTS_MODEL_OPUS` — открыть Agents Hub и:

### Тест 1: Workflow (все агенты)
В разделе Workflow создать задачу:
- Заголовок: `Тест команды`
- Запрос: `Нужна мини-воронка для продажи консультации по AI-контенту за 5000 руб через Instagram`

Ожидание:
- 7 шагов в workflow (CEO → Operations → Funnel → Content Strategy → Rewriter → Tech Architect → CEO final)
- Каждый агент даёт ответ по своей роли
- CEO финальный собирает единый результат
- Нет сырого Markdown `##` или `**` в UI

### Тест 2: Чат с Chief (Opus)
В разделе Агенты выбрать **Chief / Главный** и написать:
`Что мне сделать сегодня, чтобы быть ближе к первой продаже?`

Ожидание:
- Ответ чёткий, без воды
- Максимум 3 приоритета
- NEXT ACTION в конце
- Отвечает быстро и умнее, чем раньше — это Opus, а не Sonnet

### Тест 3: Чат с Content Maker
Написать: `Напиши мне пост для Instagram — личный опыт с AI-инструментом, который сэкономил мне время. Стиль живой, CTA — написать "схема" в комментарии.`

Ожидание:
- Готовый текст поста без `##` и `**`
- Хук в первых 2–3 словах
- CTA конкретный
- Нет GPT-запаха

### Тест 4: Чат с Analyst
Написать: `Разбери вирусную карусель в Instagram по теме AI или нейросети — вытащи механику хука и структуру.`

Ожидание:
- Разбор по шаблону: хук / структура / эмоция / CTA
- Вывод: что Егор может взять прямо сейчас

## Что менять в промптах

Каждый `.md` файл в `prompts/` содержит:
- Роль и задачу агента
- Контекст проекта Егора (продукты, воронка, ограничения)
- Правила работы
- Формат ответа

Безопасно менять:
- Добавлять конкретные кейсы и факты, когда они появляются
- Уточнять правила под текущий фокус
- Обновлять продуктовую лестницу при запуске нового продукта

Не менять:
- Стиль-эталон Егора — он зафиксирован и выверен
- Анти-GPT и антиплагиат фильтры в copywriter.md — они критичны
- Фильтр "7–14 дней до денег" в chief.md — это главный антихаос-механизм

## Brain Log — как записывать

Каждое важное решение записывать через кнопку «Записать в мозг» в Agents Hub.

Типы записей:
- `decision` — принятое решение
- `insight` — важное наблюдение
- `worked` — что сработало
- `not_worked` — что не сработало
- `task` — задача на будущее
- `note` — заметка / контекст

Тег `[BRAIN LOG]` в ответе агента = сигнал что это нужно сохранить.
