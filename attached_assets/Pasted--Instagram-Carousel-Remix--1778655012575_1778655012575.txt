КОРОТКАЯ СУТЬ ЗАДАЧИ

Нужно доработать уже существующую плитку Instagram Carousel Remix, не создавая новый пайплайн и не ломая текущее рабочее пространство.

Новая логика такая:

Instagram URL
→ import через Apify
→ AI analysis
→ rewrite
→ сборка готового ТЗ / prompt pack для ChatGPT
→ ты вручную идёшь в ChatGPT, прикладываешь style reference и вставляешь prompt
→ получаешь готовые слайды
→ загружаешь готовые слайды обратно в приложение
→ приложение публикует карусель в Instagram

NanoBanana / fal.ai из основного пользовательского flow убираем.
Генерацию картинок внутри приложения не делаем.
Основная генерация теперь ручная через твой оплаченный ChatGPT.

2. ЧТО ТЫ ИМЕЕШЬ В ВИДУ НА ЧЕЛОВЕЧЕСКОМ ЯЗЫКЕ

Ты уже сделал важную часть:

импорт карусели;
анализ слайдов;
OCR;
видишь, что на слайдах написано.

Теперь тебе не нужен дорогой автогенератор изображений внутри сервиса.

Тебе нужен инструмент, который:

вытаскивает структуру чужой карусели;
переписывает текст;
собирает готовое, сильное ТЗ для ChatGPT;
даёт его тебе в удобном виде: копировать / скачать;
потом принимает уже готовые изображения, которые ты сделал вручную в ChatGPT;
и публикует их в Instagram-карусель.

То есть приложение превращается из “AI image generator” в “carousel remix + prompt builder + publisher”.

3. КОРНЕВАЯ ЦЕЛЬ ЗАДАЧИ

Сделать из текущего Instagram Carousel Remix рабочий production-friendly инструмент, где:

импорт работает как сейчас;
анализ становится умнее;
rewrite остаётся;
вместо встроенной генерации изображений появляется Prompt Pack для ChatGPT;
после ручной генерации появляется загрузка готовых слайдов;
затем появляется официальная публикация карусели в Instagram.

type SlideAnalysis = {

  sourceImageUrl: string | null;

ОБЩИЕ ПРАВИЛА:
- Формат каждого слайда: вертикальный Instagram carousel, 4:5.
- Каждый слайд должен быть отдельным полноценным изображением.
- Все слайды должны быть в одном стиле.
- Сохраняй точный смысл текста.
- Не добавляй лишние факты, которых нет в исходном тексте.
- Не меняй названия сервисов, брендов и людей.
- Не сокращай важные названия.
- Если текста много, аккуратно структурируй его в блоки, списки, карточки или колонки.
- Если на слайде список инструментов, сделай его читаемым: номер, иконка/логотип, название, короткое описание.
- Если слайд является обложкой, сделай его максимально цепляющим: крупный заголовок -хук, который воспринимается так, будто это раскрывается какой то секрет, и сейчас будешь мясо полезной информации, или какая то жесткая польза будет, сильный визуальный образ, минимум лишнего.
- Если слайд является финальным CTA, сделай его максимально понятным и конверсионным: крупный вопрос/обещание, действие, ключевое слово для комментария.
- Используй синий цвет для смысловых акцентов: цифры, ключевые слова, CTA, номера слайдов.
- Не перегружай слайды мелким текстом.
- Текст должен быть крупным, читаемым и аккуратно выровненным.
- Не делай мультяшный, детский или шаблонный дизайн.
- Не используй случайные декоративные элементы.
- Не делай дизайн в стиле презентации PowerPoint.
- Не делай кислотные цвета.
- Не делай дешёвые 3D-эффекты.
- Не добавляй водяные знаки.
