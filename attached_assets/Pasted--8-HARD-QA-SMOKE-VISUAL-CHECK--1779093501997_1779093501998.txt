ДОБИВОЧНЫЙ ПРОХОД 12 — BACKEND HARDENING
Что можно усилить

Backend уже рабочий по структуре, но есть места для укрепления:

appendBrainLog сейчас принимает tags как string[] по типу, но runtime-жёсткой проверки массива строк в brain.ts нет. Надо добавить нормализацию tags.
readErrorMessage в brainApi.ts сначала пытается response.json(), потом response.text(). Если body уже consumed, fallback может потерять текст не-JSON ошибки. Можно улучшить через response.clone().
/wb/health пока не показывает anthropic readiness. Сейчас health показывает openrouter/fal/threads/apify, но для Agents Hub полезно добавить:
anthropic: !!process.env.ANTHROPIC_API_KEY
anthropicModelCeo: !!process.env.ANTHROPIC_MODEL_CEO
anthropicModelExecutor: !!process.env.ANTHROPIC_MODEL_EXECUTOR
Можно добавить GET /wb/brain/health, но не обязательно.
Делать отдельным проходом, не сейчас

Потому что это уже не “завершить MVP”, а hardening.
