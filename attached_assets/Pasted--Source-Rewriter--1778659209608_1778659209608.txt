Source Rewriter

Назначение:

Файл / видео / аудио / PDF / презентация / картинка / текст
→ извлечь весь контент

- редактор изображений

2. ГЛАВНОЕ ПРАВИЛО ДЛЯ АГЕНТА

Агенту нельзя додумывать.

По умолчанию:

rewriteMode = preserve_original_structure

Это значит:

Если загружена презентация → результат по слайдам.
Если загружен PDF → результат по страницам / секциям.
Если загружено видео → результат по транскрипту / сценам.
Если загружено аудио → результат по транскрипту.
Если загружено изображение → OCR + визуальное описание + переписанный текст.
Если загружен текст → тот же текст, переписанный в моём стиле.

Нельзя автоматически превращать всё в Instagram-пост.

12. EXTRACTION LOGIC
Важно

Не обещать невозможное.

Агент должен реализовать то, что поддерживается текущим проектом и доступными пакетами. Если для конкретного типа файла нет готовой поддержки, endpoint должен вернуть controlled warning, а не падать.

12.1. Text / MD

Для .txt и .md:

1. прочитать buffer как utf-8
2. нормализовать переносы
3. вернуть fullRawText
4. fileType = text
12.2. DOCX

Для .docx:

1. извлечь текст
2. сохранить заголовки/абзацы насколько позволяет extractor
3. вернуть fullRawText
4. fileType = docx

Если extractor не установлен:

return ExtractedSource with:
fullRawText = ""
extractionWarnings = ["DOCX extraction is not configured yet."]

Не падать.

12.3. PDF

Для .pdf:

Минимальный обязательный уровень:

1. извлечь текст
2. сохранить fullRawText
3. если page-level extraction доступен — заполнить pages
4. если page-level extraction недоступен — pages = []
5. добавить warning, если page-level недоступен

Не делать сразу сложный PDF renderer, если его нет в проекте.

Визуалы PDF:

Если в проекте уже есть renderer PDF page → image:
- отрендерить страницы
- отправить страницы в vision
- сохранить visualAssets

Если renderer нет:
- не добавлять новый тяжелый renderer в первом проходе
- вернуть warning:
"PDF visual extraction is not configured yet. Text extraction completed."
12.4. PPTX

Для .pptx:

Если в проекте есть PPTX extractor:
- извлечь slide-by-slide text
- вернуть slides[]

Если extractor нет:
- вернуть warning:
"PPTX extraction is not configured yet."

Важно: не пытаться парсить PPTX руками regex-ом.

12.5. Image

Для .png/.jpg/.jpeg/.webp:

1. отправить изображение в vision model
2. получить visibleText
3. получить visualDescription
4. получить styleDescription
5. получить clothing
6. получить accessoriesAndProps
7. получить lighting
8. получить background
9. получить composition
10. получить colors
11. получить recreationNotes
12. fullRawText = visibleText
13. visualAssets = [result]
12.6. Audio

Для .mp3/.wav:

Если в проекте уже есть transcription provider:
- отправить audio на transcription
- получить transcript
- fullRawText = transcript
- transcript = transcript

Если transcription provider нет:
- вернуть warning:
"Audio transcription is not configured yet."

Не добавлять новый платный сервис без отдельного решения.

12.7. Video

Для .mp4/.mov:

Если в проекте есть video/audio helper:
- извлечь audio
- отправить audio на transcription
- transcript = result

Если video frame extraction есть:
- взять 3–7 кадров
- отправить кадры в vision
- visualAssets = frame descriptions

Если frame extraction нет:
- не добавлять тяжелый video processing слой
- вернуть warning:
"Video frame analysis is not configured yet."

Главный минимум для видео:

transcript

Визуал — второй уровень.

Создать отдельную константу:

const USER_STYLE_PROMPT = `...`;

Но лучше вставить уже этот рабочий смысл:

Ты пишешь в стиле маркетолога-практика.

Стиль:
- живо
- просто
- человечески
- без академичности
- без инфоцыганства
- без GPT-воды
- без длинных заходов
- без стерильной экспертности
- без фраз “в современном мире”, “важно понимать”, “давайте разберёмся”
- короткие абзацы
- один абзац = одна мысль
- связные переходы
- конкретика
- примеры
- внутренние мысли
- честные наблюдения
- лёгкая ирония, если уместно
- без пафоса
- без умничанья

Текст должен звучать так, будто живой человек взял исходную мысль, пропустил через свой опыт и объяснил по-человечески.

Нельзя:
- копировать чужие формулировки
- делать синонимайзинг
- сохранять чужой тон
- делать школьный пересказ
- писать как методичка
- писать как корпоративный блог
- писать как GPT
- выдумывать факты

17. LONG INPUT HANDLING

Если fullRawText слишком длинный для модели:

Агент не должен слепо отправлять всё в один prompt.

Правильная логика:

1. Если текст помещается в лимит модели — отправить целиком.
2. Если не помещается — разбить на chunks.
3. Для каждого chunk сделать rewrite отдельно.
4. Собрать chunks обратно в исходном порядке.
5. Не делать summary вместо rewrite, если пользователь не выбрал summary/shorter.

Минимальная реализация:

chunkSize = безопасный размер по символам
chunkOverlap = 500 символов

Для MVP можно использовать character-based chunking.

Запрет:

ПРОХОД 7. EXTRACTION RULES

Text / MD:
- read buffer as utf-8
- normalize line breaks
- return fullRawText
- fileType = text

DOCX:
- use existing extractor if available
- if not available, return warning:
"DOCX extraction is not configured yet."
- do not crash

PDF:
- extract text
- return fullRawText
- preserve pages if extractor supports page-level extraction
- if page-level unavailable, pages = []
- if visual extraction unavailable, warning:
"PDF visual extraction is not configured yet. Text extraction completed."

PPTX:
- use existing PPTX extractor if available
- if not available, warning:
"PPTX extraction is not configured yet."
- do not parse PPTX manually with regex

Image:
- send image to vision model
- extract visibleText
- create visual description
- return fullRawText = visibleText
- visualAssets = [description]

Audio:
- use existing transcription provider if available
- if not available, warning:
"Audio transcription is not configured yet."

Video:
- use existing video/audio helper if available
- transcript is required only if transcription provider exists
- frame analysis only if existing frame extraction exists
- if unavailable, warning:
"Video frame analysis is not configured yet."

ВАЖНО:
Не добавлять новые платные сервисы без отдельного решения.
Не добавлять тяжелый video/PDF renderer без подтверждения.
Если тип файла пока не поддержан, возвращать controlled warning, а не падать.
